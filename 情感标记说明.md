# 情感标记说明

## 方法概述

使用**情感词典匹配法**为每首唐诗标注情感。逐诗扫描，统计每种情感关键词的命中数，取命中数最高的情感作为该诗的标签。

## 情感词典

基于唐诗常用情感词汇构建，分 6 类：

| 情感 | 标记 | 关键词 |
|------|------|--------|
| 悲 | ① | 愁、悲、苦、泪、泣、哀、伤、痛、忧、惨、断肠、惆怅、凄凉、凄、寂、寞、孤、独 |
| 思 | ② | 思、念、忆、归、梦、怀、望、相思、忆昔、回首 |
| 喜 | ③ | 喜、欢、乐、笑、醉、赏、咏、歌、舞、悦、惬、逍遥、得意 |
| 壮 | ④ | 壮、雄、剑、马、战、塞、征、军、戈、弓、豪、气、志、侠 |
| 怨 | ⑤ | 怨、恨、愤、怜、弃、薄、妒、谗、捐、负、摧 |
| 闲 | ⑥ | 闲、静、幽、清、禅、隐、琴、鹤、钓、菊 |

选择原则：
- 只选情感倾向明确的词（如"愁""悲"），排除中性高频字（如"山""水""月""云"——它们几乎每首诗都有，无区分度）
- 每个情感类目避免过于宽泛

## 标记逻辑

```
输入: 一首诗的文字
过程:
  1. 统计每个情感类别的关键词在诗中的出现次数
  2. 取总命中数最高的情感类别
  3. 在该诗开头插入对应的单字符标记
  4. 若所有情感类别命中数均为 0，则不加标记（保持原样）
输出: 标记字符 + 原诗
```

标记字符使用 Unicode 的带圈数字（Enclosed Alphanumerics, U+2460-U+2465）：
- ① (U+2460) = 悲
- ② (U+2461) = 思
- ③ (U+2462) = 喜
- ④ (U+2463) = 壮
- ⑤ (U+2464) = 怨
- ⑥ (U+2465) = 闲

## 执行代码

```python
from collections import Counter

EMOTIONS = {
    '悲': ['愁', '悲', '苦', '泪', '泣', '哀', '伤', '痛', '忧', '惨',
           '断肠', '惆怅', '凄凉', '凄', '寂', '寞', '孤', '独'],
    '思': ['思', '念', '忆', '归', '梦', '怀', '望', '相思', '忆昔', '回首'],
    '喜': ['喜', '欢', '乐', '笑', '醉', '赏', '咏', '歌', '舞', '悦',
           '惬', '逍遥', '得意'],
    '壮': ['壮', '雄', '剑', '马', '战', '塞', '征', '军', '戈', '弓',
           '豪', '气', '志', '侠'],
    '怨': ['怨', '恨', '愤', '怜', '弃', '薄', '妒', '谗', '捐', '负', '摧'],
    '闲': ['闲', '静', '幽', '清', '禅', '隐', '琴', '鹤', '钓', '菊'],
}

MARKS = {'悲': '①', '思': '②', '喜': '③', '壮': '④', '怨': '⑤', '闲': '⑥'}

with open('data/peot.txt', encoding='utf-8') as f:
    lines = f.read().split('\n')

tagged = []
for poem in lines:
    scores = {}
    for emo, kws in EMOTIONS.items():
        score = sum(poem.count(kw) for kw in kws)
        if score > 0:
            scores[emo] = score
    if scores:
        top_emo = max(scores, key=scores.get)
        tagged.append(MARKS[top_emo] + poem)
    else:
        tagged.append(poem)  # 无情感标记，保持原样

with open('data/peot.txt', 'w', encoding='utf-8') as f:
    f.write('\n'.join(tagged))
```

## 与模型训练的关系

标记字符直接写入 `peot.txt`，是纯文本的一部分。`process.py` 的 `get_data()` 在构建词表时自动将这些字符纳入词表，无需额外修改。

训练时，每首诗的结构为：
```
SOP → ① → 诗 → EOP
```

模型学到的是：当情感标记为 ① 时，后续内容多为悲伤风格。
生成时，先输入 SOP，再输入 ①，模型就会在悲伤风格的上下文中继续生成。

## 已知局限

1. **关键词命中次数取 Top-1**：多情感的诗只取一种标签，可能丢失部分情感信息。例如"感时花溅泪，恨别鸟惊心"同时含"泪"（悲）和"恨"（怨），只标为悲。
2. **情感词典不覆盖所有情况**：咏物诗、叙事诗可能没有情感关键词（19.3% 无标记）。
3. **未考虑否定词**："不愁""无悲"等否定结构会被误判为正面命中。
4. **一词多义**："可怜"在唐诗中多作"可爱"解，但"怜"同时出现在"怨"类中。

## 改进方向

- 使用预训练语言模型（如 BERT）做情感分类，精度更高
- 改为多标签分类，一首诗可对应多种情感
- 引入否定词检测，避免"不愁"被算作"愁"
- 按体裁调整关键词权重（如边塞诗自动倾向"壮"）
