> 版本：一天可实现版  
> 日期：2026-07-12  
> 目标：引入 THUAIPoet 数据，将当前字符级双层单向 LSTM 改为可按体裁和关键词生成的字符级 Decoder-only Transformer，并完成数据、训练、评价、实验报告与 PPT 闭环。


## 二、数据来源与用途

### 当前唐诗数据

- 补充唐诗语料及五律、七律候选；
- 修复重复、旧缓存和错误数据划分；
- 保留合法生僻字和少量真实 `□`。

### THU-CCPC

- 作为五绝、七绝的主要扩充数据；
- 使用官方 train、valid、test；
- 使用题目、作者、朝代和 1～4 个自动关键词；
- 支持关键词条件生成。

CCPC 正文用 `|` 分句，官方数据为绝句，并已提供训练、验证和测试集合。[THU-CCPC](https://github.com/THUNLP-AIPoet/Datasets/tree/master/CCPC)

### THU-CRRD

- 提供平声字、仄声字和平水韵 30 个韵部；
- 当天用于生成后平仄、押韵评价；
- 时间允许时再用于解码过滤。

CRRD 官方说明资源尚不完整，因此未知字必须单独报告，不能直接判错。[THU-CRRD](https://github.com/THUNLP-AIPoet/Datasets/tree/master/CRRD)

### THU-FSPC 与 PQED

- FSPC 有整首和逐句五级情感人工标签，留作后续情感控制；
- PQED 留作后续质量评价器或候选排序器；
- 两者不阻塞当天主链路。[THU-FSPC](https://github.com/THUNLP-AIPoet/Datasets/tree/master/FSPC)

THUAIPoet 数据仅供学术使用，报告、README 和 PPT 必须注明来源并引用论文。[数据集总览](https://github.com/THUNLP-AIPoet/Datasets)

## 四、预处理要求

### 4.1 文本规范化

1. 严格按 UTF-8 读写；
2. 去空行和首尾空白；
3. 统一中英文常用标点；
4. 使用 `|` 保存句界；
5. 去除明显标题混入、括号注释、数字和英文说明；
6. 保留合法汉字和真实 `□`；
7. 不直接删除生僻字；
8. 保留原始数据，不覆盖源文件。

### 4.2 体裁标注

```text
4句且每句5字 → WUJUE
4句且每句7字 → QIJUE
8句且每句5字 → WULV
8句且每句7字 → QILV
其他          → OTHER
```

约定：

- 标点不计入字数；
- `□` 计一个字位；
- CCPC 已知为绝句，只区分五言/七言；
- 当前唐诗数据得到的是形式体裁弱标签；
- 每类随机抽查至少 20 首。

高置信训练样本必须：句数正确、句长完全一致、无空句、无英文数字和明显注释、`□` 占比不超过 20%。

### 4.3 跨库去重与防泄漏

规范化正文后按以下优先级处理：

```text
CCPC test 中的诗 → 从所有训练和验证来源移除重复项
CCPC valid 中的诗 → 从所有训练来源移除重复项
其余训练重复     → 只保留一条并记录来源
```

不能把 CCPC 官方三组数据合并后重新随机划分。

当前唐诗数据单独按 80%/10%/10% 划分，再与 CCPC 对应集合合并。合并后再次检查三个集合之间无精确重复。

### 4.4 PLACEHOLDER 与 UNK

```text
原始□              → PLACEHOLDER
词表外低频真实字符 → UNK
```

- `PLACEHOLDER` 可作为输入；
- `target == PLACEHOLDER` 时忽略损失；
- 生成时禁止 `PLACEHOLDER`；
- `PAD`、`SOP` 不得在正文输出；
- `UNK` 默认禁止输出；
- `EOP` 必须允许输出。

### 4.5 预处理验收

| 指标 | 标准 |
|---|---|
| UTF-8 严格读取 | 成功 |
| 空记录 | 0 |
| train/valid/test 精确重复 | 0 |
| 每类体裁数量 | 已统计 |
| 每类人工抽查 | ≥20首 |
| `□` 与特殊字符数量 | 已统计 |
| 跨库删除重复数量 | 已统计 |
| 数据来源 | 每条可追踪 |

## 五、总体模型架构

### 5.1 当前基线

```text
字符 Embedding → 双层单向 LSTM → Linear(vocab_size)
```

保留作基线说明。只有修复数据划分后重新训练的结果才能用于正式比较。

### 5.2 新模型

采用字符级 Decoder-only 因果 Transformer：

```text
体裁 token + 关键词字符 + SOP + 正文
                   ↓
Token Embedding + Position Embedding
                   ↓
2～4 层因果 Transformer Block
                   ↓
LayerNorm → Linear(vocab_size)
                   ↓
下一字符 logits
```

第一版参数：

```text
d_model=128（稳定后可改256）
num_layers=2（稳定后可改4）
nhead=4（稳定后可改8）
ffn_dim=512（稳定后可改1024）
dropout=0.1
max_seq_len=128
AdamW，lr=3e-4，weight_decay=0.01
gradient_clip=1.0
```

### 5.3 输入格式

```text
<GENRE_QIJUE>
<KW_BEGIN>月<KW_SEP>故乡<KW_END>
<SOP>正文<EOP>
```

关键词内部按汉字编码，只增加少量结构 token，不为每个关键词建立新 token。

特殊 token：

```text
PAD、UNK、SOP、EOP、PLACEHOLDER
GENRE_WUJUE、GENRE_QIJUE、GENRE_WULV、GENRE_QILV、GENRE_OTHER
KW_BEGIN、KW_SEP、KW_END
```

### 5.4 新模型必须实现

#### 训练

1. 输入 `[batch, seq_len]`；
2. causal mask 禁止看到未来；
3. padding mask 屏蔽 PAD；
4. 下一个字符预测；
5. 忽略 PAD 和 PLACEHOLDER 目标；
6. 输出训练/验证平均损失；
7. 保存 best/latest 检查点；
8. 保存词表、特殊 token 和配置；
9. 支持恢复训练；
10. 固定随机种子。

#### 推理

1. 指定五绝或七绝；
2. 输入 0～4 个关键词；
3. 可选输入开头；
4. 必须支持 Greedy；
5. 时间允许时支持 temperature、top-k、top-p；
6. 禁止 PAD、SOP、PLACEHOLDER、默认禁止 UNK；
7. 正文未完成时禁止提前 EOP；
8. 强制句数、句长和标点；
9. 输出统一 JSON；
10. 时间允许时支持多候选。

### 5.5 格式约束

五绝：`5字，5字。5字，5字。` 之后 EOP。  
七绝：`7字，7字。7字，7字。` 之后 EOP。

解码过滤顺序：

1. 禁止特殊 token；
2. 正文位置禁止标点；
3. 标点位置只允许规定标点；
4. 完成前禁止 EOP；
5. 最后标点后只允许 EOP；
6. 时间允许时加入重复惩罚；
7. 执行 Greedy 或采样。

## 六、训练与实验

### 6.1 训练前必须小样本过拟合

- 使用 20～100 首；
- loss 必须明显下降；
- 模型应能复现或接近训练样本；
- 保存加载后输出一致；
- 未通过则不启动正式训练。

### 6.2 实验顺序

1. **E0 链路验证**：Transformer-small + 小数据 + Greedy；
2. **E1 主实验**：完整数据 + 体裁/关键词条件；
3. **E2 解码对比**：Greedy、top-k、top-p，时间允许时；
4. **E3 LSTM基线**：在正确划分上重训，时间允许时。

### 6.3 必须记录

- 参数量；
- 训练/验证数据量；
- 训练和验证 loss；
- 最佳 epoch/step；
- 训练耗时与硬件；
- 解码参数；
- 各体裁生成案例；
- 自动与人工评价结果。

### 6.4 训练交付

```text
checkpoints/best.pt

configs/train_config.json
```

## 七、评价方案

### 7.1 当天是否需要新评价模型

**不需要，也不建议。**

当天采用：

```text
规则脚本评价 + 小规模匿名人工评价
```

原因：格式、字数、标点、关键词、重复、平仄和韵部可以直接计算；新评价模型需要额外数据、训练和校准，一天内难以证明可靠。PQED评价器放到后续。

### 7.2 自动评价必须实现

#### 格式

- 句数正确率；
- 每句字数正确率；
- 标点正确率；
- 体裁格式完全正确率；
- EOP 正常结束率。

#### 特殊 token

- PAD、SOP、PLACEHOLDER、UNK 泄漏率。

#### 关键词

- 输入关键词数；
- 完整出现的关键词数；
- 精确关键词覆盖率。

#### 重复与记忆

- 连续字符重复；
- 重复句；
- 二元/三元组重复率；
- 与训练集完全相同的生成比例；
- 时间允许时计算最高文本相似度。

#### 平仄与押韵

使用 CRRD：

- 平仄字典覆盖率；
- 已知位置平仄匹配率；
- 未知/两可字符数量；
- 偶数句韵脚所属韵部；
- 韵脚是否同韵；
- 韵部覆盖率。

未知字符不能直接判错。

### 7.3 人工评价

匿名随机排序，每组至少 10～20 首，四人共同评分：

| 维度 | 分值 |
|---|---:|
| 通顺度 | 1～5 |
| 古诗风格 | 1～5 |
| 关键词相关性 | 1～5 |
| 上下句连贯性 | 1～5 |
| 整体意境 | 1～5 |
| 明显复制训练诗 | 是/否 |

当天不评价用典，不强行给起承转合语义打分。
